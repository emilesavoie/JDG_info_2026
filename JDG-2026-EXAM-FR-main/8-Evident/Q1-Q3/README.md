# Q1-Q3 - Analyse UX du prototype d'inspection (2 + 2 + 4 points)

**Objectif :** Évaluer de manière critique l'expérience utilisateur du prototype et identifier les obstacles potentiels.

## Contexte

Le prototype fictif présenté ici est une petite application web permettant d'enregistrer des inspections, visualiser des métriques simples et recevoir des recommandations automatiques. Le code est volontairement simple (HTML/CSS/JS vanilla) et contient plusieurs problèmes d'ergonomie à analyser.

## Exécution

- Ouvrez `index.html` dans un navigateur (double-clic) ou servez le dossier via un serveur local si nécessaire.
- Aucune installation n'est requise.

## Q1 - Charge cognitive (2 points)

**Objectif :** Analysez l’interface du prototype et identifiez deux éléments qui pourraient augmenter la charge cognitive des utilisateurs. Proposez une solution pour réduire cette charge tout en conservant les fonctionnalités.

**Livrable :** Dans le cahier d'examen

## Q2 - Dark patterns (2 points)

**Objectif :** Expliquer ce qu’est un Dark pattern dans le domaine du UI/UX. Identifier un dark pattern dans l’application Justifiez pourquoi c’est un dark pattern et comment vous feriez pour le régler.

**Livrable :** Dans le cahier d'exmamen

## Q3 - Lacunes et améliorations UX (4 points)

**Objectif :** Vous devez:

- Identifiez au moins 3 lacunes ou points de friction dans l’expérience utilisateur de l’outil.
- Pour chaque lacune faites un lien avec les principes d’utilisabilité de Tognazzini (ci dessous) et comment ceux-ci ne sont pas suivis.
- Ensuite proposez une amélioration concrète par lacune, en justifiant comment elle réglerait celle-ci.

**Livrable :** Dans le cahier d'examen
